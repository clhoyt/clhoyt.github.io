Tags and Wranglers¶

Tags are a custom data-type that help track and organize the various PI attributes, their data, and their units. The Wrangler is a sort of container than holds multiple tags.

Tag¶

class radical.dataIO.PI.wranglers.Tag(name, units=None, data=None)¶

This is an object that enables the organized packaging of a pulled attribute’s data and units. Typically, the DataProxy will receive data in a JSON format. The proxy object can convert to a dataframe and store the data and units for each attribute it pulls from PI into a Tag object for safekeeping and transfer.

Attributes:

name (str) : A name for the tag. This should typically be the actual attribute name from PI.

units (str): The units of the data, e.g. “m” or “ft”. An exception will be raised if units not listed in the available units attribute are provided.

data (dataframe): Should contain a “Timestamp” column and a “Values” column.

Wrangler¶

class radical.dataIO.PI.wranglers.Wrangler¶

This is a class that stores and manipulates Tag objects. Simplifies the process of working with large groups of Tags.

convert_units(convert_to='imperial', use_tags=None)¶

This method will run through all the tag objects, verify their current units, and then convert them to the desired units. All tags will be converted if use_tags is left to default None.

get_tidy_df(use_tags=None)¶

Assembles a ‘tidy’ dataframe for analysis.

Args:
use_tags (list): Optional, a list of tags to use to assemble the tidy dataframe
Returns:
A time-aligned, tidy dataframe.
set_alias_using_mapper(mapper)¶

This method attaches the standard name that the Detector objects expect to reference. This name is applied to the alias attribute. For example, “HookloadActual” name would get an alias name of “hook_load”.

verify_units_system_uniformity()¶

Method verifies that the units for all tags are metric or imperial, but not both. ValueError is raised if units are mixed.
